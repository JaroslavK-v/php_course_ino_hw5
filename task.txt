+Данная домашняя работа будет похожа на предыдущие (2 и 3), но должна быть реализована
 с использованием фреймворка Laravel и предоставляемых им инструментов 
(а точнее тех, о которых было рассказано на лекции 12 и 14).

Необходимо реализовать функционал регистрации/аутентификации пользователей
(с определенными валидационными правилами для каждого из полей: например ограничения на минимальное
 количество символов в пароле, на уникальность логина, на правильность формата электронной почты).
 НЕ ИСПОЛЬЗУЕМ Auth из коробки Laravel.

Минимальный набор полей пользователя: логин, пароль, емайл, фамилия+имя, пол (select),
 мобильный телефон.

Аутентификация может быть как через логин, так и через емайл (приветствуется работа
 с сессиями через хелпер https://laravel.com/docs/5.4/session).

Текущие пользователи должны хранится в отдельном json файле, новые должны записываться 
в конец файла (допускается хранение в бд, работа с бд достаточна в виде конструктора запросов).

Также для пользователей должна быть реализована страница отображения информации о них.

Должна быть возможность установить аватарку пользователя 
(адрес к загруженной аватарке необходимо хранить в файле с остальными данными)

Кроме роли обычных пользователей должна быть реализована роль администратора. 
Он может просматривать список всех пользователей, их страницы.

Необходимо максимально воспользоваться возможностями роутинга в Laravel.

Для отображения страниц использовать шаблоны (наследуемые от главного), 
подключить стили bootstrap для минимально лаконичного отображения стандартных блоков html.

Дополнительные задания:

Реализация кастомного класса Request с необходимыми валидационными правилами
Использование кастомных сообщений об ошибках при валидации
